# InfraQuest — 260 Bricks

A single-file interactive curriculum: 260 daily lessons (52 weeks, Mon–Fri) taking you from zero to job-ready Infrastructure / Cloud / Platform Engineer. Every brick on the wall is one day — tap it to see what to learn, exact tasks and commands, the proof to save, the beginner mistake to avoid, the small win, and a Quran/hadith quote.

Progress (bricks laid, tasks ticked, hours invested) saves automatically in your browser via localStorage. Use **export/import progress** at the bottom to move progress between phone and laptop.

## Put it live on GitHub Pages (5 minutes, works from your phone)

1. Go to **github.com → New repository**. Name it `infraquest` (or anything). Public. Create.
2. On the repo page: **Add file → Upload files** → upload `index.html` (and this `README.md`). Commit.
3. **Settings → Pages** → under "Build and deployment": Source = **Deploy from a branch**, Branch = **main**, folder = **/ (root)** → Save.
4. Wait ~1 minute. Your app is live at:
   `https://YOUR-USERNAME.github.io/infraquest/`
5. Open that URL on your phone → browser menu → **Add to Home Screen**. Now it sits next to your other apps.

## Notes

- Everything is inside `index.html` — no build step, no dependencies, no backend. Editing a lesson = editing the JSON inside that file.
- Progress is per-browser. Clearing browser data clears progress — export a backup at milestones (end of each month is a good habit).
- The plan: Months 1–2 Linux + Bash · 3 Networking · 4 Git + Python · 5 AWS · 6 Docker · 7 CI/CD + Scaling · 8 Terraform · 9–10 Kubernetes · 11 Observability + Security + AI infra · 12 Capstone + Interviews · Weeks 49–52 Job hunt + deepening.

Brick by brick. Bismillah.
